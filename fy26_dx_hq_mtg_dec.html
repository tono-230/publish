01 Service Data

AIで音声データを分析し、
顧客理解と改善の起点にする。

CSだけではなく、店頭での接客も音声データとして蓄積し、顧客が何を求め、どこで迷い、どの体験に満足したのかを把握する。

POINT 01

音声データを資産化

店頭接客やCS対応を音声データとして保存し、ニーズ・不満・質問を抽出して、個人の経験に閉じない知見へ変換する。

POINT 02

顧客体験を横断把握

購入、保証、修理、問い合わせを一つの流れで見える化し、顧客ごとの次回提案やフォローに活用する。

POINT 03

スタッフのスキルアップ・商品開発・CRM

良い接客の型、よくある不満、売れる提案を抽出し、スタッフのスキルアップ・商品開発・CRMに展開する。

目指す状態

店舗とCSで発生するサービス接点を、音声を含めて継続的にデータ化し、接客品質の向上、顧客満足度改善、リピート促進につながる改善サイクルを作る。

02 Operation Data

AIでオペレーションデータを分析し、
店舗運営を最適化する。

お客様の動線とスタッフの動線をデータで捉え、可視化・予測・アクション提案までを行う店舗オペレーションAIソリューションを開発する。

POINT 01

動線・業務量を可視化

お客様とスタッフの動き、受付・測定・加工・受け取りなどの業務量をデータで捉え、店舗の状態を見える化する。

POINT 02

混雑・負荷を予測

時間帯・来店状況・スタッフ配置・作業待ちをもとに、混雑や業務負荷の偏りを事前に予測する。

POINT 03

次アクションを提案

役割分担、優先順位、応援判断、案内タイミングなど、店舗が次に取るべきアクションをAIが提案する。

目指す状態

現場の感覚に頼っていた店舗運営を、データに基づく可視化・予測・アクション提案へ変え、繁忙店でも少人数で安定したサービス品質を維持する。

03 Location Data

AIでロケーションデータを分析し、
売場改善、在庫の最適化につなげる。

RFID・棚割・商品位置・在庫状態をつなぎ、売場の現状を可視化。AIが売上・在庫・陳列位置を分析し、棚割変更や補充などの売場改善実行につなげる。

POINT 01

売場の状態を可視化

RFIDと棚割情報を接続し、商品がどこにあり、どの棚がどう使われているかをデータで把握する。

POINT 02

AIが売場課題を分析

売上・在庫・陳列位置・滞留状況を組み合わせ、売れ筋や機会損失、棚効率の課題を見つける。

POINT 03

売場改善を実行

棚割変更、商品配置、補充、展開強化など、店舗が次に取るべき売場改善アクションへつなげる。

04 Face Data / 05 Frame Data

顔と商品を3Dで捉え、
提案品質とフィッティング精度を高める。

フェイスデータとフレームデータを3Dで保持し、スタッフのフィッティングスキルを底上げ。パーソナルデータの可視化により、お客様の納得感と信頼感を高める。

04 フェースデータ

顔型、顔幅、鼻位置、耳位置、フィット感、似合う傾向、VTO利用履歴を3Dデータとして保持し、個人ごとのフィッティングガイドに活用する。

05 フレームデータ

形状、サイズ、カラー、素材、重量、売れ筋、3Dモデルを整理し、フェイスデータと掛け合わせて提案・フィッティング・商品開発に接続する。

USE CASE

フィッティングスキル底上げ

顔と商品属性を掛け合わせたフィッティングガイドを出力し、スタッフごとの調整品質を底上げする。

USE CASE

パーソナルデータを可視化

お客様自身の顔型・サイズ感・似合う理由を見える化し、提案への納得感と信頼感を醸成する。

USE CASE

提案・商品開発へ接続

蓄積したフェイス／フレームデータを、接客提案、VTO高度化、商品開発、ECでの選びやすさに活用する。

Locker Example

参考：JINS 銀座店 ロッカー設置例
ISSUE 01

複数台設置によるスペース占有

後日受け取り分までロッカーで対応しようとすると、保管量に応じて複数台の設置が必要になり、店舗スペースを大きく占有する。

ISSUE 02

当日分中心では効果が限定的

現状のOWNDAYSロッカー運用は当日渡しが中心のため、スタッフの受け取り対応工数削減という本来のメリットを十分に享受できていない。

DIRECTION

高密度な自動保管へ

当日分だけでなく後日受け取り分まで集約できる高密度な自動保管により、省スペース化と受け取り対応の省人化を両立する。

2026年7月 DX推進課 本部会議資料13
Solution Introduction

ロッカーよりも、
高効率・高密度な自動保管へ。

完成品を自動で収納し、受け取り時も自動で引き渡すことで、受け取り対応を省人化。高密度・高効率な保管により、当日渡しだけでなく後日受け取り分まで限られた店舗スペース内で管理する。

ロッカー以上の高密度保管

完成品をケース単位で高密度に管理し、当日渡し・後日受け取り分をまとめて保管できる導線を構築する。

QRで自動呼び出し

お客様がQRをスキャンすると、SpiderPICK機構が対象商品を取り出し、受取口まで搬送する。

接客を止めない受取

スタッフが接客・測定・加工中でも受取対応を止めず、後日受け取りの待ち時間と店舗負荷を下げる。

検証ポイント

ロッカー比での保管密度、省スペース性、取り出し精度、店舗オペレーションとの接続、顧客体験、導入コストをPoCで確認し、実店舗展開の可否を判断する。
