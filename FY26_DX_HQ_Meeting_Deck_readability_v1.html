01 Service Data

音声データを取得し、
顧客理解と改善の起点にする。

CSだけではなく、店頭での接客も音声データとして蓄積し、顧客が何を求め、どこで迷い、どの体験に満足したのかを把握する。

POINT 01

接客内容を資産化

店頭接客やCS対応を音声データとして保存し、ニーズ・不満・質問・提案内容を抽出して、個人の経験に閉じない知見へ変換する。

POINT 02

顧客体験を横断把握

購入、保証、修理、問い合わせを一つの流れで見える化し、顧客ごとの次回提案やフォローに活用する。

POINT 03

教育・商品・CRMへ接続

良い接客の型、よくある不満、売れる提案を抽出し、スタッフ教育・商品開発・マーケティングに展開する。

目指す状態

店舗とCSで発生するサービス接点を、音声を含めて継続的にデータ化し、接客品質の向上、顧客満足度改善、リピート促進につながる改善サイクルを作る。

2026年7月 DX推進課 本部会議資料08
02 Operation Data

AIでオペレーションデータを分析し、
店舗運営を最適化する。

来店、受付、測定、加工、受取、在庫、スタッフ動線をデータで捉え、AIが混雑予測・役割分担・優先順位・次アクションを提案できる運営基盤へつなげる。

POINT 01

業務量を予測する

来店数、測定待ち、加工件数、受取件数を時間帯別に把握し、AIが繁忙予測をもとに人員・役割・準備の最適化を支援する。

POINT 02

ボトルネックを特定する

受付から受取までの滞留時間や作業待ちを可視化し、AIがどの工程で体験が止まっているか、どこを改善すべきかを示す。

POINT 03

AI推奨アクションへ

混雑時の役割変更、測定誘導、加工優先順位、受取対応などをAIがリアルタイムに提案し、店長・スタッフ判断を支援する。

目指す状態

現場の感覚に頼っていた店舗運営を、AIとデータに基づく予測・配置・改善へ変え、繁忙店でも少人数で安定したサービス品質を維持する。

2026年7月 DX推進課 本部会議資料09
03 Location Data

AIでロケーションデータを分析し、
売場改善、在庫の最適化につなげる。

商品、棚、什器、売場レイアウト、RFID、Planogramをつなぎ、今どの売場に何が並び、何が出し切れていないかをAIで分析し、売場改善と在庫最適化につなげる。

POINT 01

売場の現状を把握

店舗ごとの什器配置、棚位置、商品展開状況をデータ化し、在庫過剰・過少の可視化、理想のロケーションを本部と店舗で共有する。

POINT 02

RFIDと棚割を接続

スキャン結果と計画棚割を比較し、欠品・過剰在庫・配置ズレ・滞留商品を早期に検知し、売場在庫の過剰や未陳列を改善する。

POINT 03

売場改善を実行へ

日々の販売実績や売場データを分析し、本部・SV・店長が最適な売場と商品配置をプランニングできる状態を作り、改善指示を店舗タスクに落とし込む。

目指す状態

Planogramと日々の実績をつなぎ、売場づくり、在庫探索、欠品対応、過剰在庫是正、発注精度、売上改善に使えるロケーション基盤を構築する。

2026年7月 DX推進課 本部会議資料10
04 Face Data / 05 Frame Data

顔と商品を3Dで捉え、
提案品質とフィッティング精度を高める。

顔型・サイズ感・似合う傾向と、フレームの形状・サイズ・カラー・3Dモデルを組み合わせ、顔のパーツに合ったフィッティングガイドや接客提案につなげる。

04 フェースデータ

顔型、顔幅、鼻位置、耳位置、フィット感、似合う傾向、VTO利用履歴を蓄積し、顔のパーツに合わせたフィッティングガイド出力や個人に合った提案に活用する。

05 フレームデータ

形状、サイズ、カラー、素材、重量、売れ筋、3Dモデルを整理し、VTO、フィッティング提案、商品開発、MDに接続する。

USE CASE

フィッティング精度向上

顔と商品属性を掛け合わせ、顔のパーツに合わせたフィッティングガイドを出力し、接客品質を均質化する。

USE CASE

VTO高度化

3Dデータで自然な試着体験とフィット感の再現性を高め、ECでも安心して選べる体験へつなげる。

USE CASE

教育・EC活用

スタッフ全体の提案レベル向上や、ECでのフィット不一致による返品率削減につなげる。

Solution Introduction

ロッカーよりも、
高効率・高密度な自動保管へ。

完成品をメガネケースごとロッカーよりも高密度に保管し、QRスキャンを起点に対象商品を自動で取り出す。受取対応の省人化と、限られた店舗スペースの有効活用を同時に狙う。

ロッカー以上の高密度保管

完成品をケース単位でまとめて管理し、従来ロッカーよりも高密度・高効率な保管と受取導線を構築する。

QRで自動呼び出し

お客様がQRをスキャンすると、SpiderPICK機構が対象商品を取り出し、受取口まで搬送する。

接客を止めない受取

スタッフが接客・測定・加工中でも受取対応を止めず、後日受取の待ち時間と店舗負荷を下げる。

検証ポイント

ロッカー比での保管密度、省スペース性、取り出し精度、店舗オペレーションとの接続、顧客体験、導入コストをPoCで確認し、実店舗展開の可否を判断する。
